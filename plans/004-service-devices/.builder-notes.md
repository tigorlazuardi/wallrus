## Critical: DeviceFilterCriteria shape mismatch

The IMPLEMENTATION.md spec describes `res_min`, `res_max`, `aspect`, `size_bytes` fields. However, `src/lib/server/db/schema.ts` (the ground truth for what's in SQLite) defines `DeviceFilterCriteria` as:

```ts
type DeviceFilterCriteria = {
  min_width?: number
  max_width?: number
  min_height?: number
  max_height?: number
  aspect_ratio?: { target: number; tolerance: number }
  min_bytes?: number
  max_bytes?: number
  formats?: Array<"jpg" | "png" | "webp" | "avif">
  tags_include?: string[]
  tags_exclude?: string[]
  nsfw: "all" | "sfw_only" | "nsfw_only" // NOTE: required, not optional
}
```

The `DeviceFilters.ts` Zod schema MUST match this shape, not the spec's description. `nsfw` has a default of `"all"` in the Zod schema (required field → `.default("all")`).

## Cursor encoding

- Cursor: base64url-encoded JSON `{ created_at: number, id: string }`
- `encode_cursor`: JSON.stringify → Buffer → base64url (no padding)
- `decode_cursor`: returns null on any parse error (never throws)
- `limit`: clamped to [1, 200], default 50
- `next` and `prev` mutually exclusive; `next` wins

- Original used `AppError.is(err, AppError) ? err : AppError.wrap(...)` — `AppError.is` returns `T | undefined` not boolean, so `err` in the truthy branch was still `unknown`. Fixed using nullish coalescing: `AppError.is(err, AppError) ?? AppError.wrap(err, ...)`.

## Route test placement — CRITICAL

SvelteKit's build manifest scanner rejects any file inside `src/routes/` that starts with `+` unless it's a real route. This means `+server.test.ts` files CANNOT live alongside the route files. They were moved to:

- `src/test/routes/api/v1/devices/server.test.ts`
- `src/test/routes/api/v1/devices/slug/server.test.ts`
- `src/test/routes/api/v1/devices/slug/toggle/server.test.ts`

Imports use deep relative paths (e.g., `../../../../../routes/api/v1/devices/+server`). This is the only option since there's no `$routes` alias. The `[slug]` directory becomes `slug` in the test path since `[` and `]` are problematic in filenames.
