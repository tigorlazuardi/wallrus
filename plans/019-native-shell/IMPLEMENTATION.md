# 019 — Native shell: Capacitor + native wallpaper plugin + distribution

## Status

**deferred (human-gated — NOT for Ralph loop).** This slice's
verification gates require a native toolchain (Android SDK / Gradle,
Xcode on macOS), an emulator or physical device, and human sign-off
(TestFlight acceptance, on-device wallpaper change). An automated loop
**cannot** green-gate these — do NOT attempt to drive this slice with
`/ralph-loop`. It is excluded from the `WALLRUS-MVP-COMPLETE` promise
index check (see `plans/README.md`).

Depends on [`016-mobile-shell`](../016-mobile-shell/) being **done**
(all mobile-shaped TS + the static adapter build must exist first).

## Goal

Wrap the static `build-mobile/` output in Capacitor on Android + iOS,
implement the native `WallpaperPlugin` whose TS contract was defined in
016, and ship internal-distribution builds (self-hosted APK +
TestFlight) so the daemon becomes a phone-first wallpaper-set surface.

## Decisions (pre-baked)

### Inherits 016 contracts verbatim

- TS plugin interface: `src/lib/client/mobile/wallpaper.ts`
  (`registerPlugin("Wallpaper")`) — the native side registers under the
  same `"Wallpaper"` name.
- Setup/login, Bearer storage, boot logic, release-manifest schema, and
  the "Set as wallpaper" UI all already exist in `src/lib/` from 016.
  This slice adds NO new web behavior — only the native wrapper +
  implementations + release pipeline.

### Single repo, NO monorepo — Capacitor in `mobile/`

```
wallrus/
├── build-mobile/                   # static adapter output (from 016)
├── mobile/
│   ├── capacitor.config.ts         # webDir: "../build-mobile"
│   ├── android/                    # generated by `cap add android`
│   ├── ios/                        # generated by `cap add ios`
│   └── plugins/wallpaper/
│       ├── android/WallpaperPlugin.kt
│       └── ios/WallpaperPlugin.swift
```

### Capacitor scaffold

- `bun add @capacitor/cli @capacitor/android @capacitor/ios` (runtime
  libs `@capacitor/core` + `@capacitor/preferences` already added in 016).
- `bun x cap init wallrus io.wallrus.app --web-dir=build-mobile`.
- `bun x cap add android` + `bun x cap add ios`.
- `mobile/capacitor.config.ts` — CSP permits `capacitor://localhost`.

### Native wallpaper plugin

- **Android**: download bitmap (OkHttp, native-side — avoid base64 bridge)
  → `WallpaperManager.getInstance(ctx).setBitmap(bitmap, null, true, flags)`
  where `flags = FLAG_SYSTEM | FLAG_LOCK` per `target`. Declare
  `SET_WALLPAPER` in `AndroidManifest.xml` (install-time, no runtime prompt).
- **iOS**: Apple forbids programmatic set. Download (URLSession) → save to
  Photos (`PHPhotoLibrary.shared().performChanges`) → present
  `UIActivityViewController`. Return
  `{ success: true, note: "saved-to-photos-share-required" }` (the 016 UI
  already shows the explainer for this note).
- Register in `MainActivity.java` (Android) and `AppDelegate.swift` /
  `Plugin.m` (iOS).

### Distribution

- **Android**: self-hosted APK served via the 016
  `/api/v1/mobile/release/latest` endpoint (homelab fit, no store fees).
  Signing keystore gitignored, path/passwords via env.
- **iOS**: TestFlight (only legal internal path — no self-hosted iOS).
- App Store / Play Store public listing is post-MVP.

### Out of scope

- Public store listings, offline cache, push, multi-daemon, widgets,
  Tauri path (post-MVP).

## Resume here

### Phase 1 — Capacitor scaffold

1. `bun add @capacitor/cli @capacitor/android @capacitor/ios`.
2. `bun x cap init wallrus io.wallrus.app --web-dir=build-mobile`.
3. `bun x cap add android`, `bun x cap add ios`.
4. `mobile/capacitor.config.ts` — CSP for `capacitor://localhost`.
5. `bun x cap sync` succeeds; emulator/sim boot and land on `/setup`.

### Phase 2 — native wallpaper plugin

6. `mobile/plugins/wallpaper/android/WallpaperPlugin.kt` (WallpaperManager
   + OkHttp download) + `SET_WALLPAPER` in `AndroidManifest.xml` +
   register in `MainActivity.java`.
7. `mobile/plugins/wallpaper/ios/WallpaperPlugin.swift` (save-to-Photos +
   share-sheet, URLSession download) + register in `AppDelegate.swift`.
8. Device/emulator verification: wallpaper actually changes on Android;
   save + share-sheet appears on iOS.

### Phase 3 — release wiring

9. Android: adaptive icon, splash, signing keystore (env-driven),
   `SET_WALLPAPER` confirmed; build signed APK; publish to the
   `/api/v1/mobile/release/latest` source.
10. iOS: bundle ID, provisioning profile, icon, splash; TestFlight build.
11. `mobile/README.md` — build + sign + distribute for both platforms.
12. First internal APK published + first TestFlight build accepted.

## Gotchas / Deferred

- **macOS + Xcode required for iOS.** Android can be built on Linux with
  the Android SDK; iOS cannot. Plan device access accordingly.
- **iOS wallpaper limitation**: programmatic set forbidden — save + share
  is the only legal flow. The 016 UI already surfaces this before the tap.
- **`SET_WALLPAPER`**: normal install-time permission, no runtime prompt.
- **CSP**: permit `capacitor://localhost` in `capacitor.config.ts`.
- **Native download**: plugin downloads the image itself (OkHttp / URLSession)
  — never pass huge base64 across the JS/native bridge.
- **Tauri 2 alternative**: still on the table if Capacitor webview perf
  disappoints. Not in scope here.
