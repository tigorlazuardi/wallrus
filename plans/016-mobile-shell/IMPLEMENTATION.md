# 016 — Mobile shell: dual adapter + Capacitor + native wallpaper plugin

## Status

**not-started (deferred)** — do NOT start until the user greenlights
mobile work explicitly. This slice depends on
[`015-shared-ui`](../015-shared-ui/) being **done** first.

## Goal

Wrap the static SvelteKit build in Capacitor on Android + iOS. Add a
custom native plugin so users can tap a wallpaper in the gallery and
have it actually set as the device wallpaper (Android: programmatic
via `WallpaperManager`; iOS: save-to-Photos + share-sheet per Apple's
allowed flow). Ship internal-distribution builds (APK + TestFlight)
so the homelab daemon can be a phone-first wallpaper-set surface
without leaving the existing SvelteKit web app behind.

## Decisions (pre-baked)

### Builds on top of slice 015

This slice assumes slice 015 has shipped:

- Pages use `+page.ts` universal load via `/api/v1/*`.

### Single repo, NO monorepo

no Turborepo. Justification recorded in chat: solo dev, single-user
homelab, Capacitor needs only `webDir` pointing at the static
build — restructure tax > benefit.

### Dual adapter build

```js
// svelte.config.js
import adapterBun from "svelte-adapter-bun"
import adapterStatic from "@sveltejs/adapter-static"
import { vitePreprocess } from "@sveltejs/vite-plugin-svelte"

const useStatic = process.env.WALLRUS_ADAPTER === "static"

export default {
  preprocess: vitePreprocess(),
  kit: {
    adapter: useStatic
      ? adapterStatic({
          fallback: "index.html", // SPA fallback for client router
          pages: "build-mobile",
          assets: "build-mobile",
        })
      : adapterBun(),
  },
}
```

```json
// package.json
{
  "scripts": {
    "build": "bun --bun vite build",
    "build:mobile": "WALLRUS_ADAPTER=static bun --bun vite build"
  }
}
```

### First-launch setup screen

Mobile bootstrap reads `@capacitor/preferences` key `api_base`:

```ts
// mobile/src/app.ts
import { Capacitor } from "@capacitor/core"
import { Preferences } from "@capacitor/preferences"
import { set_api_base } from "$lib/client/config"
import { goto } from "$app/navigation"

export async function boot_mobile() {
  if (!Capacitor.isNativePlatform()) return

  const { value } = await Preferences.get({ key: "api_base" })
  if (value) {
    set_api_base(value)
    return
  }
  await goto("/setup")
}
```

`/setup` route (gated by `Capacitor.isNativePlatform()`):

- Text input for daemon URL.
- "Test connection" button — `fetch(${url}/healthz)` expects 200.
- On success: `Preferences.set({ key: "api_base", value: url })` →
  `set_api_base(url)` → `goto("/")`.

Same screen handles re-config later (e.g. moved daemon, new server).

### Token storage

Mobile uses Bearer JWT (cookie auth is browser-only). Login flow:

```ts
const { access_token } = await useAuthMutation().login({ username, password })
await Preferences.set({ key: "auth_token", value: access_token })
```

`fetcher.ts` reads token via:

```ts
const { value } = Capacitor.isNativePlatform()
  ? await Preferences.get({ key: "auth_token" })
  : { value: null } // web uses cookie
if (value) headers.set("Authorization", `Bearer ${value}`)
```

If auth is disabled on the daemon (`WALLRUS_AUTH_ENABLE=false`,
post-013), token step is skipped. Setup screen surfaces both states.

### Native wallpaper plugin contract

```ts
// mobile/plugins/wallpaper/src/index.ts
import { registerPlugin } from "@capacitor/core"

export interface WallpaperPlugin {
  setWallpaper(opts: {
    imageUrl: string // local file:// or http(s)://
    target: "home" | "lock" | "both" // Android only
  }): Promise<{ success: boolean; note?: string }>
}

export const Wallpaper = registerPlugin<WallpaperPlugin>("Wallpaper")
```

- **Android**: download bitmap → `WallpaperManager.getInstance(ctx).setBitmap(bitmap, null, true, flags)` where `flags = FLAG_SYSTEM | FLAG_LOCK | FLAG_SYSTEM` per `target`.
- **iOS**: Apple does NOT allow programmatic wallpaper set. Save to Photos (`PHPhotoLibrary.shared().performChanges`) → present `UIActivityViewController` with the saved asset. User taps "Use as Wallpaper" from the share sheet. Return `{ success: true, note: "saved-to-photos-share-required" }` so the UI can show the explainer.

### "Set as wallpaper" UI

Wired into the image detail screen. Behavior:

- Android: dropdown picker (Home / Lock / Both) → calls plugin → toast on success.
- iOS: explainer dialog ("iOS doesn't let apps change wallpaper directly. We'll save the image and open the share sheet — tap 'Use as Wallpaper'.") → calls plugin → share sheet appears.

### Distribution

Decision point at end of slice — defer until phase 7:

- **Self-hosted APK** via `/api/v1/mobile/release/latest` (homelab-friendly, no store fees, no review). Auto-update via version check at boot.
- **Play Store internal track** (more legitimate, requires Play Console account).
- **TestFlight** for iOS internal (no choice — no self-hosted iOS).

Recommend: self-hosted APK for Android (homelab fit), TestFlight for iOS (only legal path), App Store + Play Store post-MVP if ever public.

## Gotchas / Deferred

- **Slice 015 dependency is hard.** Mobile reuse breaks if pages
  still call `+page.server.ts` direct-service. Confirm 015 done
  before starting phase 4.
- **iOS wallpaper limitation**: programmatic set forbidden. Save +
  share is the only legal flow. Surface this in UI BEFORE the user
  taps the button — surprise costs trust.
- **`SET_WALLPAPER` permission**: declare in
  `mobile/android/app/src/main/AndroidManifest.xml`. Granted at
  install time (normal permission, no runtime prompt).
- **CSP**: Capacitor webview serves from `capacitor://localhost`
  (Android) or `capacitor://localhost` (iOS). Adjust SvelteKit CSP
  if applied; permit those origins in `capacitor.config.ts`.
- **SSE in Capacitor**: tested-OK on Android WebView ≥116 and iOS
  WKWebView. No polyfill needed.
- **Single-instance assumption**: mobile app talks to ONE daemon
  URL per install. Multi-daemon is post-MVP, deferred.
- **`$lib/schemas/*` from mobile-only code**: SvelteKit alias
  resolves inside the webview build. Pure mobile/plugins code
  outside SvelteKit build needs path alias or relative imports —
  decide in phase 5.
- **Capacitor preferences ≠ secure**: `@capacitor/preferences` is
  SharedPreferences / NSUserDefaults — readable on rooted/jailbroken
  devices. Acceptable for homelab; if upgraded to public app, swap
  to `@capacitor-community/secure-storage`.
- **Wallpaper download**: plugin receives a URL; downloads itself
  on the native side to avoid passing huge base64 across the JS/native
  bridge. Use `OkHttpClient` (Android) / `URLSession` (iOS).
- **Tauri 2 mobile alternative**: still on the table for later if
  Capacitor webview perf disappoints (smaller binary, native render,
  Rust core). Not in scope here.
