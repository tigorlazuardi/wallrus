### Key: testing .svelte.ts files in bun:test

`$state` is a Svelte 5 rune ‚Äî NOT available in bun:test without Svelte compiler.
Polyfill with `(globalThis as any).$state = <T>(v: T): T => v` BEFORE importing the module.
Use dynamic import (`await import(...)`) to ensure the polyfill is in place when the
module initialiser runs. Import path must NOT include `.ts` extension
(`"$lib/client/devices/use-devices.svelte"` not `...svelte.ts`).

The mutation hook has no `$state` so it can use static import directly.

### Prefer-const rule

ESLint enforces `const` for `$state` declarations (the variable is never reassigned,
only its properties are mutated). Use `const state = $state(...)` not `let state = $state(...)`.
