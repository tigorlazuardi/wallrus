### Key: testing .svelte.ts files in bun:test

`$state` is a Svelte 5 rune — NOT available in bun:test without Svelte compiler.
Polyfill with `(globalThis as any).$state = <T>(v: T): T => v` BEFORE importing the module.
Use dynamic import (`await import(...)`) to ensure the polyfill is in place when the
module initialiser runs. Import path must NOT include `.ts` extension
(`"$lib/client/devices/use-devices.svelte"` not `...svelte.ts`).

The mutation hook has no `$state` so it can use static import directly.

### Prefer-const rule

ESLint enforces `const` for `$state` declarations (the variable is never reassigned,
only its properties are mutated). Use `const state = $state(...)` not `let state = $state(...)`.

### superforms SPA pattern for Phase 3

Template for any `+page.ts` universal load + SPA form:

```ts
// +page.ts
import { superValidate } from "sveltekit-superforms"
import { zod4 as zod } from "sveltekit-superforms/adapters"
import { SomeRequestSchema } from "$lib/schemas/<domain>/SomeOp"

export const load: PageLoad = async ({ fetch, params }) => {
  const res = await fetch(`/api/v1/<domain>/${params.slug}`)
  if (!res.ok) error(res.status, `...`)
  const item = SomeResponseSchema.parse(await res.json())
  const form = await superValidate(
    {
      /* initial data */
    },
    zod(SomeRequestSchema),
  )
  return { form, item }
}
```

```svelte
<!-- +page.svelte -->
const { form, errors, enhance, submitting, message } = superForm(data.form, {
  dataType: "json",
  SPA: true,
  validators: zodClient(SomeRequestSchema),
  onUpdate: async ({ form, cancel }) => {
    if (!form.valid) return
    try {
      const result = await useSomeMutation().create(form.data)
      await goto(`/<domain>/${result.slug}`)
    } catch (e) {
      form.message = (e as Error).message
      cancel()
    }
  },
})
```

### zod4 adapter (reminder for Phase 3)

Server-side load: `import { zod4 as zod } from "sveltekit-superforms/adapters"` and use `zod(Schema)`.
Client-side validators: `import { zod4Client as zodClient } from "sveltekit-superforms/adapters"` and use `zodClient(Schema)`.

### fetch in universal load

Use the `fetch` from SvelteKit's `load()` params — NOT `apiFetch`. The SvelteKit-provided `fetch` handles cookies/credentials correctly for both SSR and client-side navigation. `apiFetch` is for mutation hooks and client-side-only fetches.
