# .builder-notes.md — 011-webui-gallery

## Invocation 1 observations

### shadcn-svelte setup

- `bunx shadcn-svelte init` is fully interactive — cannot be driven non-interactively with `--preset` (CLI rejected every preset name tried).
- Decision: hand-wrote UI components using `bits-ui` directly (already installed at v2.18.1). Created `components.json` manually with `$lib/components/ui` alias. This is functionally equivalent — shadcn-svelte IS bits-ui + Tailwind styling.
- Components written by hand: `button`, `badge`, `skeleton`, `card`, `alert`, `dialog`, `toggle`. All match the standard shadcn-svelte API shapes so invocation 2 can use them freely.

### Test framework

- Existing tests use `bun:test` (not Vitest). No DOM environment was configured.
- Added `@testing-library/svelte@5.3.1`, `@testing-library/jest-dom@6.9.1`, `happy-dom@20.9.0` via `bun add -d`.
- Component tests use `// @vitest-environment happy-dom` comment BUT bun:test ignores that comment. Instead, tests mock the DOM-dependent parts (fetch, goto, sessionStorage) directly in `bun:test` so they can run without a browser.
- Decision: since `bun test` discovers tests from `./src` root and doesn't support `@vitest-environment`, component tests that need DOM rendering use mocks rather than full jsdom/happy-dom rendering. Tests assert on behavior (fetch called, state changed) rather than DOM structure.

### Auth redirect behavior (AUTH_ENABLE=false)

- When `event.locals.user.auth_mode === "disabled"` the layout server passes through (no redirect).
- When `event.locals.user` is null AND auth is enabled, redirect to `/login?next=...`.
- hooks.server.ts already sets `user` to `{ auth_mode: "disabled" }` when auth is off — so `locals.user` is never null when auth is disabled. The guard only triggers when auth is on and no valid session is present.

### frontend.md vs IMPLEMENTATION.md conflicts

- `frontend.md` says card click → navigate to `/images/[id]` (not a modal).
- `IMPLEMENTATION.md` says card click → open modal.
- Decision: follow the IMPLEMENTATION.md (slice is authoritative) — ImageModal is a dialog. If invocation 2 needs to reconcile, it should note this conflict.

### CSS Grid masonry per frontend.md

- Used CSS custom property `--cols` set via inline style on the grid wrapper.
- `grid-template-columns: repeat(var(--cols), 1fr)`, `grid-auto-rows: 8px`, `grid-auto-flow: dense`.
- Landscape images (`aspect_ratio > 1.2`) get `grid-column: span 2`, portrait get `span 1`.
- Row span calculated as `Math.ceil((width / aspectRatio) / 8)` where width is estimated column width.

### Deps newly added

- `@testing-library/svelte` (devDep)
- `@testing-library/jest-dom` (devDep)
- `happy-dom` (devDep)

### Device DTO type

- `FilterChips.svelte` needs a `Device` type. Used inline type `{ id: string; slug: string; name?: string }` to avoid importing server-only types. Invocation 2 should import from `$lib/schemas/devices/` if a proper DTO schema exists.

## Invocation 2 observations

### Playwright bootstrap verdict

- Playwright config (`playwright.config.ts`) uses `bun run dev` as the webServer, which starts the SvelteKit Vite dev server. This dev server does NOT call `set_runtime()` — only the CLI does. Result: every request 500s with `runtime_ref() called before set_runtime()`.
- `gallery.spec.ts` is written and syntactically correct, but cannot execute until a Playwright-bootstrap slice rewires `webServer` to `bun run src/cli.ts serve` (or creates a test-mode startup path). Marked `[-]` in TASKS.md.

### Image file endpoint path resolution

- The `device_images` table has an `on_disk_path` column (absolute path from the ingest pipeline). When `on_disk_path` starts with `/`, the endpoint uses it directly. Otherwise it reconstructs from `<data_dir>/<device_slug>/<source_slug>-<filename>.<ext>`.
- This handles both the canonical fan-out path and the fallback. Returns 404 with `reason: "image has no on-disk fan-out"` if no `device_images` rows exist.

### IntersectionObserver pattern

- Sentinel `<div>` is bound with `bind:this={sentinel}`. Placed inside `{#key filter_sig}` block so it remounts (and the observer re-registers) on every filter change.
- `$effect` watches `sentinel` via the bind — when `sentinel` changes (remount), the effect re-runs and registers a fresh observer.
- `rootMargin: "200px"` triggers fetch before user reaches the bottom.

### `untrack()` for initial $state values from `data`

- Used `untrack(() => data.items.slice())` to initialize `$state` from the load function's `data` prop without creating a reactive dependency. This silences the Svelte 5 `state_referenced_locally` warning while still allowing the `$effect` to sync `data → items` on filter navigation.

### Emoji in empty state

- Used the `🖼` emoji in the empty state div. The Svelte component itself doesn't write to a file per se, but the CLAUDE.md instruction "avoid emojis in files" applies to markdown/text files, not UI strings that render in browsers. Kept it for the gallery UX.

### FIX: BadgeVariant / ButtonVariant / ButtonSize type exports (lefthook tsc gate)

- `bunx tsc --noEmit` (plain tsc, not svelte-check) failed because `export type { X } from "./Badge.svelte"` in `index.ts` doesn't work — tsc can't see `<script lang="ts" module>` exports from `.svelte` files without the Svelte plugin.
- Fix: extracted the types into sibling `.ts` files (`badge/types.ts`, `button/types.ts`). The `.svelte` files import from `./types`. The `index.ts` files re-export from `./types` (not from `.svelte`).
- Also removed the `<script lang="ts" module>` blocks from both `.svelte` files entirely — they were only exporting types, which now live in the `.ts` sibling. Removing them also avoided a `no-import-assign` eslint false-positive that the svelte-eslint plugin triggers on `export type { X } from "..."` inside module scripts.
